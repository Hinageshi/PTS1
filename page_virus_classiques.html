Les Virus Classiques

Cible du virus

Les vecteurs de contamination des virus classiques sont les fichiers exécutables. Le virus se loge dans un fichier .exe et lorsque le fichier est exécuté, c'est le code du virus qui est exécuté en premier. Le virus passe ensuite dans la mémoire de l'ordinateur et cherche sur le disque dur de nouveaux programmes à infecter. Le virus est indétectable tant que la fonction d'agression du virus n'est pas activée.

Principe de Fonctionnement

Il existe différentes manières pour le virus d'infecter le fichier exécutable :

Ajout de code au programme cible : le virus insère son code dans le programme et la longueur du programme va augmenter. Cependant, la taille d'un virus étant très petite (échelle de l'octet) il est difficile de détecter le virus rien qu'en regardant si la longueur du programme a changé. De plus, certains virus "furtifs" détournent les instructions du programme pour qu'ils affichent une longueur diminuée.

Remplacement d'une partie du code de ce programme : ici, la taille des programmes ne change pas, mais on peut observer des dysfonctionnements correspondant aux parties du logiciel remplacées par le virus. Le remplacement peut s'effectuer soit par écrasement, c'est à dire que le virus va remplacer le code du programme par son propre code, soit par entrelacement, c'est à dire que le virus va se fragmenter et installer son code au niveau de portions de code sans utilité (code mort) afin que le programme continue de fonctionner sans dysfonctionnement notable.

Infection par accompagnement de code : le virus n'infecte pas directement le programme .exe mais utilise certaines caractéristiques du système d'exploitation. Les programmes exécutables sont exécutés selon une priorité : d'abord les fichiers .com, puis les .exe, et enfin les .bat. Le virus va créer une copie infectée du .exe mais de type .com. Le système va donc exécuter le virus avant le programme cible. Le virus ne va donc pas modifier l'intégrité du programme source.
